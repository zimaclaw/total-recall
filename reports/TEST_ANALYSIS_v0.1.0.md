## 🔍 КЛЮЧЕВЫЕ НАБЛЮДЕНИЯ

### 1. **Одинаковые результаты для разных вопросов**

**Проблема:** Flashback по категории возвращает **ВСЕ инсайты** этой категории, а не релевантные по смыслу вопроса.

**Пример:**
```
Тест 1: "Могу ли я использовать 2 разных endpoints?"
  → Результат: Curator — это не я...

Тест 2: "Как правильно организовать бэкап данных?"
  → Результат: Curator — это не я... (ОДИНАКОВО!)
```

**Причина:** `--flashback --category "infra"` возвращает ВСЕ инсайты категории infra, без семантического поиска.

## 💡 ВЫВОДЫ

### ✅ Что работает:
1. **Flashback по категории работает** — возвращает данные из Neo4j
2. **Все 11 категорий доступны** — нет ошибок
3. **Иерархия сохраняется** — conclusion/lesson/principle/meta
4. **Конфиденциальность** — все данные локально

### ⚠️ Проблемы:
1. **Нет семантического поиска** — возвращает ВСЕ инсайты категории, а не релевантные по смыслу
2. **Одинаковые результаты** для разных вопросов в одной категории
3. **Нет ранжирования** по релевантности к вопросу

## 📝 РЕКОМЕНДАЦИЯ

**Вариант 1: Добавить семантический поиск через Qdrant**

**Причины:**
1. **Релевантность** — результаты будут соответствовать смыслу вопроса
2. **Разные ответы** — разные вопросы → разные результаты
3. **Ранжирование** — top результаты по релевантности

**Реализация:**
1. Использовать `--flashback --focus "вопрос"` вместо `--category`
2. Qdrant для векторного поиска
3. Фильтр по категории
4. Ранжирование по score
