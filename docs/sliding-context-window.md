# Сборное окно контекста

*Концепт. OpenClaw Personal Assistant Architecture.*

## Суть

Сборное окно контекста — это способ подачи сессии модели. Не память, не опыт.
Решает проблему compaction без потери информации.

## Проблема которую решает

Линейное окно контекста подаёт модели фиксированный хвост сессии:

```
[====старое (сжимается)====][==recent (подаётся)==]
                                       ↑
                                  здесь модель
```

Старое либо теряется, либо сжимается с потерей деталей.
Модель не знает что было в начале сессии если оно выпало из окна.

## Идея

Вместо линейного хвоста — сборное окно из двух частей:

```
[промпты пользователя] + [фокус — релевантный кусок сессии]
```

**Промпты** — скелет сессии. Только сообщения пользователя, без ответов.
Лёгкие, их можно брать все. Дают модели понимание что хотели
на протяжении всей сессии — без потери деталей.

**Фокус** — подвижная область. Определяется текущей задачей.
Не фиксированный хвост, а релевантный кусок сессии —
может быть из середины если текущая задача к нему относится.

## Что даёт

- Нет потерь от compaction — старое не выбрасывается, просто не в фокусе
- Контекст всегда релевантен текущей задаче
- Размер предсказуем — промпты + фокус, не "всё что влезет"
- Модель всегда видит скелет сессии целиком

## Открытые вопросы

- Как определяется фокус? Curator на основе текущего промпта решает
  какой кусок сессии релевантен и вытаскивает его?
- Как хранится сессия для произвольного доступа к любому куску?
- Граница между сборным окном и DMT — окно это контекст сессии,
  DMT это контекст из памяти. Curator собирает оба?

## Связь с архитектурой

Это ингредиент — не самостоятельный модуль. Ложится в Curator:

```
Curator сейчас:
  → flashback из памяти (DMT)
  → prepend в начало промпта

Curator с сборным окном:
  → flashback из памяти (DMT)
  → скелет сессии (все промпты пользователя)
  → фокус из сессии (релевантный кусок)
  → собирает контекст из трёх источников
```
