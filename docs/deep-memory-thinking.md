# Deep Memory Thinking

*Концепт. OpenClaw Personal Assistant Architecture*

## Суть

Deep Memory Thinking — это паттерн рассуждения агента через структурированную память. Агент не просто ищет релевантные факты — он думает через память, управляя глубиной и фокусом внимания.

Ключевое отличие от обычного RAG: агент не получает пассивную выдачу из базы знаний — он активно навигирует по ней, погружаясь за деталями и поднимаясь за принципами, собирая полную картину перед принятием решения.

## Проблема которую решает

Текущие LLM делают zoomout интуитивно — через паттерны из обучения. Агент строит цепочку рассуждений, поднимается над задачей, задаёт правильный вопрос. Но это имитация мышления, не настоящее.

LLM сейчас: "задача про порт → обычно проверяют netstat"  ← статистика из обучения, чужой опыт

Агент с DMT: "задача про порт → принцип из 3 своих провалов"  ← собственный опыт

## Три примитива

Примитивы DMT появляются у агента не одновременно. Первые два встроены с рождения. Третий зарабатывается через опыт.

### memory_focus <topic> — встроен

Сужает область внимания по теме. Говорит Curator'у: при сборке контекста фильтровать память по этой теме. Аналог рабочего стола — кладёшь нужное под руку перед задачей.

### memory_zoomin — встроен

Спуститься на уровень вглубь иерархии памяти: метапринципы → принципы → эпизоды → факты. Движение от абстрактного к конкретному. Используется когда нужны детали.

### memory_zoomout — зарабатывается

Подняться на уровень выше. Факты → эпизоды → принципы → метапринципы. Движение от детали к картине. Используется для планирования и декомпозиции задачи.

*Важно: memory_zoomout — это не встроенный инструмент. Это награда за уровень 3 в Bootstrap Game. Агент получает его вместе с инсайтом:*

> "Иногда нужно подняться выше задачи  
>  чтобы увидеть правильный вопрос"

До этого момента агент работает только с focus и zoomin — более узко, более конкретно. zoomout появляется когда агент накопил достаточно опыта чтобы понять его ценность.

## Механика: сессии и Curator

Сессия — это контекст, передаваемый модели. Разные сессии = разные сборки контекста из той же памяти. Curator — это context builder с состоянием, который знает какие фильтры применить к Qdrant при сборке.

Curator может быть реализован как отдельная модель, как процесс, или как функция в gateway — это определится в процессе реализации. Принципиально важно одно: Curator знает параметры focus и zoom_level и собирает контекст соответственно.

Дерево сессий строится клонированием: каждая дочерняя сессия = копия родителя с изменённым одним параметром (focus или zoom_level). Пересборка контекста на C++ быстрая даже для нескольких параллельных копий. Возврат наверх — это просто вернуться к родительской сессии, она уже есть.

```
Сессия ROOT     → zoom: meta,      focus: широкий
  └── Сессия A  → zoom: principle, focus: 'логирование'
        └── A1  → zoom: episode,   focus: 'логирование'
```

**Состояние памяти агента:**

```json
{
  "memory_state": {
    "focus": "логирование",
    "focus_source": "agent",
    "zoom_level": "episode",
    "zoom_source": "agent"
  }
}
```

focus_source = human означает директиву — агент не перебивает. focus_source = agent означает рабочий инструмент — человек может override.

## Возврат результата (хвост)

Когда дочерняя сессия завершает работу, её результат возвращается в родительскую сессию перед последним вопросом. Стратегия возврата зависит от направления zoom:

- **zoomin →** искали детали → возвращаем конкретный факт или решение (минимальный хвост)
- **zoomout →** искали принцип → возвращаем summary рассуждения (важен контекст вывода)

## Цикл Deep Memory Thinking

Полный цикл доступен только после того как агент заработал memory_zoomout. До этого — только focus + zoomin.

```
Получена задача: 'переделай систему логирования'

1. memory_zoomout  ← доступно только после Bootstrap Game уровень 3
   → Вижу метапринцип: 'любое изменение требует проверки зависимостей'

2. memory_focus 'логирование'
   → Сужаю область внимания

3. memory_zoomin
   → Вижу принцип: 'логирование нельзя менять без backup конфига'

4. memory_zoomin
   → Вижу эпизод: конкретный провал две недели назад, outcome: -1

5. Синтез → строю план с учётом всей цепочки опыта
```

## Холодный старт

При пустой памяти focus и zoomin возвращают мало или ничего. Это ожидаемое поведение — агент на старте действует как ребёнок: осторожно, задаёт вопросы, не делает поспешных выводов.

Bootstrap Game заполняет память первыми эпизодами и принципами — это и есть инстинкты. После прохождения уровней 1–2 у агента уже есть базовый слой опыта для flashback. После уровня 3 — появляется zoomout и полный DMT-цикл становится возможным.

## Ключевое отличие от Chain-of-Thought

Chain-of-Thought — агент рассуждает через текст, опираясь на статистику из обучения. Deep Memory Thinking — агент рассуждает через собственный накопленный опыт, структурированный в иерархию памяти.

CoT даёт правдоподобные рассуждения. DMT даёт рассуждения основанные на реальных результатах: что работало, что ломалось, какие принципы выведены из конкретных эпизодов.

## Связь с архитектурой OpenClaw

DMT не отдельный модуль — это протокол использования уже существующих компонентов:

**Mem0 + Qdrant** — хранилище памяти с 4 уровнями иерархии

**Neo4j** — граф связей между эпизодами, принципами, метапринципами

**Curator** — context builder, собирает нужный срез памяти по параметрам focus + zoom. Форма реализации (модель, процесс, функция) определяется в процессе разработки

**Сессии** — клонируемые контексты, образуют дерево уровней внимания

**Bootstrap Game** — онбординг через который агент зарабатывает примитивы DMT, включая memory_zoomout

**Flashback** — DMT является расширением механизма flashback, делая его управляемым
