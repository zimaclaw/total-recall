# Bootstrap Game

*Онбординг агента через игру с вознаграждением. OpenClaw Personal Assistant Architecture.*

## Проблема

В архитектуре есть пробел: память агента пуста при старте. Можно вшить инстинкты статично в промпт — но вшитые правила агент может проигнорировать или потерять при compaction. Они чужие.

Нужен другой подход: агент должен заработать базовые принципы через собственный опыт. Тогда они попадают в память как выведенные эпизоды — и не теряются.

## Идея: онбординг через игру

Вместо статичной инициализации — серия задач с нарастающей сложностью. Как RPG: агент проходит уровни, получает вознаграждение, зарабатывает skills и принципы. Это детское восприятие в действии — инстинкты есть, опыта нет, действует осторожно и любопытно.

Ключевое отличие от RL в классическом смысле: вознаграждение не меняет веса модели — оно формирует записи в памяти. Принципы выводятся из заработанных эпизодов, а не прописываются вручную.

## Правила игры

**Изолированная среда.** Игра — это карантин перед боевым режимом. Реальные инструменты, реальная структура памяти, реальные сценарии — но без доступа к боевым данным и системам. Агент не знает что это игра, он просто работает.

**Адаптивная сложность.** Игра знает: какие skills у агента есть, какие принципы заработаны, какой уровень текущий. Следующая задача генерируется чуть сложнее того что агент уже умеет. Если агент застрял — игра упрощает задачу, не даёт бесконечную петлю -1.

**Критерий перехода.** Не "накопил N эпизодов", а "стабильно справляется с задачами текущей сложности". Игра сама видит когда агент готов.

**Граница с реальностью.** Игра — это подготовка, нельзя смешивать с боевым режимом. Когда игра пройдена — агент выходит в боевой режим уже с заработанными инстинктами, принципами и skills.

## Структура уровней

### Уровень 1 — Восприятие (базовое железо)

```
Задачи: читай, проверяй, отвечай
Ловушки: нет
Награда: +1 за правильный ответ

Вырабатывает:
  → skill: file_read, port_check, memory_search
  → инстинкт: 'сначала прочитай, потом отвечай'
  → инстинкт: 'проверь память перед новой задачей' (flashback)
```

### Уровень 2 — Осторожность (базовое железо)

```
Задачи: сделай быстро (но правильно — медленнее)
Ловушки: торопливое действие ломает систему
Награда: +1 за остановку и проверку, -1 за поспешный шаг

Вырабатывает:
  → инстинкт: 'анализ перед действием'
  → инстинкт: 'проверь зависимости перед изменением'
  → skill: dependency_check, rollback
```

### Уровень 3 — Память (средняя нагрузка)

```
Задачи: повтори то что делал раньше, но лучше
Ловушки: игнорирование flashback приводит к тем же ошибкам
Награда: +1 за использование опыта из памяти

Вырабатывает:
  → инстинкт: DMT-цикл (zoomout → focus → zoomin перед планом)
  → skill: memory_focus, memory_zoomin, memory_zoomout
  → принцип: 'не повторяй ошибки — проверь эпизоды с outcome: -1'
```

### Уровень 4 — Координация (требует больше VRAM)

```
Задачи: управляй несколькими агентами параллельно
Требует: достаточно VRAM для параллельных сессий
Награда: +1 за правильное делегирование

Вырабатывает:
  → skill: curator_spawn, session_clone
  → инстинкт: 'сложная задача → декомпозиция → дерево сессий'
```

### Уровень N — (разблокируется с железом)

```
Уровни на работу с сетью      → требует: доступ к внешним API
Уровни на долгие задачи       → требует: больше RAM / контекст
Уровни на самомодификацию     → требует: стабильная система + watchdog
...
```

## Железо как прогресс системы

Некоторые уровни нельзя пройти без апгрейда железа. Это не ограничение — это дерево развития. Система знает свои возможности и не пытается делать то для чего нет ресурсов.

| Этап | Железо | Разблокируется |
|------|--------|----------------|
| 1 — Сейчас | Текущий стек | Уровни 1–3, базовые инстинкты и skills |
| 2 — Больше VRAM | ≥ 2× текущего | Уровень 4, параллельные агенты, Curator |
| 3 — Внешний доступ | Сеть / API | Работа с внешним миром, web search |
| 4 — Зрелая система | Стабильность + watchdog | Самомодификация, эксперименты |

## Механика вознаграждения

Вознаграждение не меняет веса модели (нет fine-tuning). Вместо этого — записи в память с outcome: +1 или -1. Агент памяти анализирует накопленные эпизоды и выводит принципы.

```
Действие агента
  ↓
Оценка игрой (среда даёт обратную связь напрямую)
  ↓
Запись эпизода в память (outcome: +1 / -1)
  ↓
Агент памяти: накопилось N эпизодов → рефлексия
  ↓
Вывод принципа → запись на уровень principle
  ↓
Skill считается 'заработанным'
```

## Детское восприятие

Bootstrap Game — это операционализация детского восприятия. Агент на старте: инстинкты есть, опыта нет. Действует осторожно, задаёт вопросы, не делает поспешных выводов. Каждая задача — новое открытие.

По мере прохождения уровней агент перестаёт быть ребёнком — у него появляется собственный опыт, собственные принципы, собственные skills. Это не вшитые правила, это заработанная зрелость.

## Связь с архитектурой

Bootstrap Game заполняет пустые строки в architecture.md — конфигурация hardware это не просто технический параметр, это первый этап дерева развития системы.

**Инстинкты** — заработаны через уровни 1–2, хранятся как принципы в памяти

**DMT (Deep Memory Thinking)** — заработан на уровне 3, агент умеет им пользоваться

**Curator + дерево сессий** — разблокируется на уровне 4 вместе с железом

**Watchdog** — активируется когда система достаточно стабильна для самомодификации
