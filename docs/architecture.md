# Архитектура персонального помощника

## Цель

Создание персонального помощника на локальных LLM, приближающегося по уровню выполняемых задач к Claude.

---

## Основные принципы

1. **Мультиагентность как средство** — не самоцель, а способ достижения результата
2. **Самообучение через память** — не дообучение весов, а накопление опыта
3. **Единая память как инструмент фокусировки** — агенты умны через взаимодействие с памятью
4. **Самовосстановление как механизм обучения** — ошибки становятся опытом
5. **Стабильность работы** — система должна быть надёжной

### Пятница (Qwen3.5-27B)
- Основной агент взаимодействия с человеком
- llamacpp на .181:8180, контекст 163K
- Работает с памятью через memory-reflect

### Ollama основной (.145:11434)
- qwen3:14b — модель для рефлексии (synthesize_principle, synthesize_meta)
- qwen2.5:7b — вспомогательные задачи

### Ollama mem0 (.145:11435)
- bge-m3 — embeddings для семантического поиска
- xitao/bge-reranker-v2-m3 — reranking результатов flashback
- qwen3:4b-mem0 — лёгкая модель для mem0

### Memory Stack
- **Neo4j** — граф памяти: Task → Evidence → Conclusion → Lesson → Principle → Meta
- **Qdrant** — векторное хранилище для семантического поиска (коллекции: memories, reflections)
- **memory-reflect** — CLI для записи дампов и flashback
- **memory-daemon** — async демон рефлексии с комбо-триггером

### Watchdog Daemon
- Отдельный процесс (изоляция от падений основной системы)
- Мониторинг здоровья основного gateway
- Откат при сбоях
- Работает с той же памятью

### Curator (Фаза 3 — в разработке)
- Внутреннее я агента: саморефлексия + неудовлетворённость
- Активный наблюдатель за ходом работы Пятницы
- Подаёт нужный контекст из памяти в нужный момент
- Начинаем с MVP: статичный сборщик контекста перед задачей

## Структура памяти

Два хранилища работают вместе.

### Neo4j — граф опыта

```
Task
  └─[HAS_EVIDENCE]──► Evidence
  └─[HAS_CONCLUSION]─► Conclusion
                          └─[GENERALIZES_TO]──► Lesson
                                                  └─[ABSTRACTED_TO]──► Principle
                                                                          └─[ABSTRACTED_TO]──► Meta
```

**Уровни иерархии:**

| Узел | Что хранит | Как создаётся |
|------|-----------|---------------|
| Task | Цель + outcome задачи | При записи дампа |
| Evidence | Факт/причина из reason | При записи дампа |
| Conclusion | Вывод из задачи + confidence (Байес) | При записи дампа |
| Lesson | Обобщённый урок | Автоматически когда confidence ≥ 0.75 |
| Principle | Принцип из кластера Lesson | memory-daemon через LLM |
| Meta | Метапринцип из кластера Principle | memory-daemon через LLM |

### Qdrant — семантический поиск

- Коллекция `reflections` — векторы выводов для семантического flashback
- Коллекция `memories` — mem0 (основная память агента)
- Embeddings: bge-m3 (1024d, Cosine)
- Reranking: bge-reranker-v2-m3

### ReflectionState

Узел в Neo4j. Хранит счётчик для триггера демона:
- `conclusions_since_last_run` — новых Conclusion с последней рефлексии
- `last_run_ts` — время последней рефлексии
- Триггер: 10 новых Conclusion ИЛИ 24 часа без рефлексии

## Flashback — как агент обращается к памяти

Перед каждым действием:

```
1. memory-reflect --flashback --category <категория>
   → возвращает иерархию: Conclusion → Lesson → Principle → Meta

2. memory-reflect --flashback --focus "тема задачи"
   → семантический поиск: vector top-20 → rerank → top-5

3. Если найден опыт с outcome: fail → учесть ошибку, поступить иначе
4. Если найден принцип → применить перед планированием
5. Если ничего → новая территория, действовать осторожно
```

## Deep Memory Thinking (DMT)

Протокол рассуждения агента через собственный опыт. Три примитива:

| Примитив | Действие | Когда доступен |
|----------|----------|----------------|
| `memory_focus <тема>` | Сузить область внимания | С рождения |
| `memory_zoomin` | Спуститься к деталям (принципы → эпизоды) | С рождения |
| `memory_zoomout` | Подняться к общему (эпизоды → принципы) | После Bootstrap уровень 3 |

Полный DMT-цикл перед сложной задачей:
```
zoomout → найти метапринцип
focus <тема> → сузить область
zoomin → найти конкретный эпизод с outcome
Синтез → план с учётом всей цепочки опыта
```

## Цепочка действий

Цепочка — последовательность шагов от запроса до результата. Основная единица опыта.

### Статусы

| Статус | Описание |
|--------|----------|
| `in_progress` | Активно работаем |
| `paused` | Отложена, можно вернуться |
| `completed` | Успех |
| `failed` | Провал (знаем почему) |
| `blocked` | Зациклились, ждём человека |
| `abandoned` | Брошена |

### Уровни автономии

| Уровень | Поведение |
|---------|-----------|
| `supervised` | Каждый шаг согласовывать с человеком |
| `autonomous` | Делай сама, доложи при проблемах или по завершении |
| `background` | Делай тихо, только логируй |

### Классификация действий по риску

| Риск | Действия | В autonomous режиме |
|------|----------|---------------------|
| `safe` | читать, проверять, искать | выполняется |
| `moderate` | менять конфиги, создавать файлы | выполняется |
| `dangerous` | удалять, перезапускать, менять права | требует подтверждения |

## Механизм самовосстановления

```
Действие
    ↓
  Сбой?
    ↓
   Да → Откат последнего действия
      → Запись в память (outcome: fail)
      → attempts += 1
      → attempts >= 30?
          ↓
         Да → Откат до стабильной точки
            → Статус: blocked
            → Эскалация: urgent
         Нет → Повторная попытка (с учётом flashback)
```

## Каналы коммуникации

| Канал | Когда | Как |
|-------|-------|-----|
| `routine` | Задача выполнена, статус по запросу | В чате, при следующем контакте |
| `normal` | Нужен ввод, решение среднего риска | В чате, инициирует сама |
| `urgent` | Зацикливание, откат до стабильной, dangerous в autonomous | Телеграм |
| `critical` | Система неработоспособна, watchdog падает | Телеграм + повторять |

## Особенности агента взаимодействия

1. **Быстрый ответ** — не тормозить человека
2. **Умный и профессиональный разговор** — качество диалога
3. **Разумное сомнение** — не уверен = спроси или проверь
4. **Сначала думаем, потом делаем** — flashback перед действием
